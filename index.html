Git is a distributed version control system.

Distributed = No central authority

Any repo can be cloned and hosted on a different computer.

You can even distribute git changes in unique ways. Sending patch files via email

Who is the administrator of your git repo clone?

You

You have the ability to modify anything in your git clone.

  • You can merge changes to master without a review
  • Rewrite commit metadata so it looks like you are the author
  • Rewrite the entire commit history of the master branch

That seems dangerous

However, you can only do this to your own repo. There are mechanisms in place so you can't push these changes to repos on github.

Under the hood git is storage system for git objects.

branches and lightweight tags are named references. Heavyweight tags are named references, but also include a second git object to identify who created the tag and when.

There is no builtin delete for git objects

objects are only deleted after being unreferenced or compressed for x days.

Git will compress git objects that need to be maintained but haven't been used for x days

The way it works is similar to the garbage collector in your favorite modern programming language.
